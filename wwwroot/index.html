Installing Svelte

  • Open your command prompt and create a new project with

    mkdir svelte-projects

  • "Now enter the folder in the command line with

    cd svelte-project

  • You need Node.js installed in order to work with Svelete's command line tools. Go to this website to download Node.js for your website: Node.js
    Make sure to download Git too, as it makes it possible to work with pre-defined templates from Svelte's support team.

  • Now that you have Git and Node.js downloaded, open a terminal in the folder you want to create your Svelte Project. Execute the following lines in the terminal

    npx degit sveltejs/template my-svelte-directory cd my-svelte-directory npm install

  • The previous command created your project inside the directory, loaded it up with critical packages, and added files from Github's sveltejs/template repository.

  • You can now run Svelte in your terminal using:

    npm run dev

What Makes Svelte Special

  • No Virtual DOM

    When using a framework that utilizes a Virtual DOM, everytime your web application updates, a new Virtual DOM is created and the old DOM is reconciled against the new one, the web application is updated. Svelte skips the overhead of this, seeing the Virtual DOM operations as additional work to be done by the web app. Svelte is instead a compiler that knows at build time, rather than run time, what will change in your app. As Richard Harris writes in this article , "...you know what would be even faster [than the additional Virtual DOM operations]? Not using them."

What We use in Nursery Online

  • Reactive Assignments

    Svelte will automatically update the DOM whenever any component's state changes. Svelte has a built in reactive operator that recomputes values whenever any relevant part changes. Take a look at the below code:

    $: doubled = number * 2;
    The "$:" signifies this as reactive declaration. Whenever the "count" variable updates, the double variable will also update.

  • Props and Components

    Svelte is built upon the idea of reusable and modular code. When designing this website, our group would load components into modules,vastly shortening not only the time it takes to write the code,but the amount of lines in the code.

    All Svelte components will be composed using the following template:

    <script>
    Insert all Business Logic here
    </script>
    Insert all HTML here
    <style>
    Insert all Styling here
    </style>

  • Next you import the component into the desired module, inside the script section:

    <import ComponentName from './pathOfComponent/Component.svelte';>

    Now whenever you need to access the component you can put it wherever it is needed within the main application. This can be on any line in your source code! It is inserted like so:

    <ComponentName />
    This will run ALL the code from the imported component. This mean you have full control of where the JavaScript, HTML, and CSS will be executed in a component based on where it is injected. But what if you need to pass in a value into a component from an external module? This is where we used Svelte Props!

  • Props in Svelte

    In order to pass a value into an external component, you must declare the variable with a name identical to the variable you wish to utilize within the component. Next, within the component, you are accessing you must add the export keyword in front of the variable:

    export let setMeOutside;

    The export keyword lets the compiler know this variable can be set outside of this component. Now return to the application using the component. Declare a variable in the script section with the same name, then pass it into the component like so:

    let setMeOutside;
    <ComponentName setMeOutside = {'This value is given to the component'}/>

  • Moving to our Web Application

    Our Web Application also heavily utilizes each loops, but thanks to Svelte we can write quick, iterative HTML code. We put each loops directly into the HTML code like so:

    {#each Array for item(item.id)}
    {/each}
    This loop can contain any HTML elements within it, allowing you to iterate through an array, all while creating appropriate HTML elements.
